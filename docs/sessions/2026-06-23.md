# Sessão — 2026-06-23

## Contexto
Projeto Nidus — plataforma full-stack open-source (tipo Appwrite + Vercel + Supabase).

## O que foi feito

### Setup inicial
- Projeto criado em `/Volumes/BACKUP/projects/nidus/`
- Estrutura mono-repo: `apps/dashboard`, `apps/control-plane`, `packages/runtime`, `packages/shared`
- Docker Compose com PostgreSQL, Redis, MinIO, Caddy (comentado GoTrue — substituído por auth próprio)
- PostgreSQL local (porta 5432) usado no lugar do Docker por instabilidade do Docker Desktop

### Control Plane (NestJS + Prisma 7)
- Auth: register, login, JWT guard (via pg pool direto — Prisma 7 model API com adapter-pg não funciona como esperado)
- PrismaService expõe `client` (PrismaClient com adapter) e `db` (pg Pool raw)
- Schema Prisma: User, Project, Deployment, Database
- Health endpoint: `GET /health`
- Auth endpoints: `POST /api/auth/register`, `POST /api/auth/login`, `GET /api/auth/me`
- Projects CRUD: `GET /api/projects`, `POST /api/projects`, `GET /api/projects/:id`
- Deployments: `GET /api/projects/:projectId/deployments`

### Dashboard (Next.js 16 + Tailwind 4)
- Tema escuro Vercel-inspired com verde como cor de destaque
- Login/Cadastro com formulário e integração com API
- Sidebar com navegação entre Projetos e Deployments
- Página de listagem de projetos (cards com status + data)
- Página de detalhes do projeto com histórico de deploys
- Página de criação de novo projeto (nome, slug, repo URL)
- API client (`src/lib/api.ts`) com JWT token management

### Runtime package
- `packages/runtime/src/index.ts`: buildProject, deployProject, stopProject
- Dockerfiles gerados para Next.js, Vaden/Dart, Express
- detectFramework() para reconhecer stacks

### Correções durante o setup
- `@prisma/adapter-pg` instalado + Pool pg para contornar incompatibilidade Prisma 7
- Docker Desktop instável (várias desconexões) → migrado para PostgreSQL local
- Porta 5433 conflitando com Docker PG → mudado para 5432 (Postgres local)
- Broto MVP (jardim-de-bolso) usando porta 3000 → parado para liberar para Nidus
- Erro "Unterminated regexp literal" no broto-mvp corrigido (extra `</div>` removido)

### Serviços rodando
- Nidus Dashboard: http://localhost:3000
- Nidus API: http://localhost:3001
- PostgreSQL: localhost:5432 (user: broto, db: nidus)

## Problemas conhecidos
1. Docker Desktop instável no macOS — desconecta frequentemente
2. Prisma 7 adapter-pg não funciona para queries model-level (user.findUnique, etc.) — usando raw SQL via pg pool como workaround
3. GoTrue abandonado — auth implementado diretamente no NestJS com JWT + bcrypt
